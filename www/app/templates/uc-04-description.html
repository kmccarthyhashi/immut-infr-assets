Tying it all togehter:



Golden images are an essential part of our immutable pattern

A golden image is essentially a template for the image-type you want to base subsequent future versions of the image off of
This allows us to have standardized policy and structure in place
We deploy Version 2, once we confirm that the changes are enacted and the new image successfully deploys, then we can decommission Version 1
We rebuild the whole thing all at once to streamline the workflow and versioning of machine images so that all images are identical and no one image has particular unique characteristics
